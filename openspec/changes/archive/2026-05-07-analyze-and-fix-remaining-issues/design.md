## 技术方案

### 1. 数据导入页面完善

**当前状态**: 占位页面，只有框架
**目标**: 完整的 Excel 导入功能

**实现步骤**:
1. 使用 `st.file_uploader` 实现文件上传
2. 使用 `pandas.read_excel` 解析 Excel
3. 数据预览（`st.dataframe`）
4. 数据验证（检查必填字段、数据类型）
5. 批量导入到 DuckDB
6. 进度显示（`st.progress`）

**关键代码**:
```python
# 文件上传
uploaded_file = st.file_uploader("上传 Excel 文件", type=['xlsx', 'xls'])

# 解析预览
if uploaded_file:
    df = pd.read_excel(uploaded_file)
    st.dataframe(df.head(10))
    
# 验证和导入
if st.button("导入数据"):
    with st.progress(0) as progress:
        for i, row in df.iterrows():
            # 验证并导入
            progress.progress((i + 1) / len(df))
```

### 2. 规划分析页面集成

**当前状态**: 占位页面
**目标**: 能选择学生并运行规划引擎

**实现步骤**:
1. 从数据库加载学生列表
2. 学生选择下拉框
3. 运行规划引擎按钮
4. 进度条显示（引擎运行可能需要时间）
5. 实时日志输出
6. 完成后跳转到结果页

**关键代码**:
```python
# 加载学生列表
students = student_service.list_students()
selected_student = st.selectbox("选择学生", students)

# 运行规划
if st.button("开始规划分析"):
    with st.spinner("分析中..."):
        routes = planning_service.generate_routes(selected_student.id)
        st.success(f"完成！生成了 {len(routes)} 条规划路线")
        st.session_state['routes'] = routes
```

## 数据流设计

### 导入流程
```
用户上传 Excel → 预览数据 → 验证数据 → 批量导入 → 更新数据库 → 显示结果
```

### 规划流程
```
选择学生 → 加载学生数据 → 运行规划引擎 → 生成路线 → 存储结果 → 展示结果
```

### 导出流程
```
选择路线 → 格式化数据 → 生成文件 → 下载
```

| 风险 | 影响 | 缓解措施 |
