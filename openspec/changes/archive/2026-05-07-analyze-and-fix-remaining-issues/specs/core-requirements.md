# 核心需求规格

## 功能需求

### FR1: Excel 数据导入
- **描述**: 支持用户上传 Excel 文件导入高校录取数据
- **验收标准**:
  - [ ] 支持 .xlsx 和 .xls 格式
  - [ ] 能预览上传的数据（前 10 行）
  - [ ] 验证必填字段（高校、专业、年份、分数）
  - [ ] 批量导入到 DuckDB
  - [ ] 显示导入进度和统计

### FR2: 规划分析执行
- **描述**: 在 UI 上运行规划分析引擎
- **验收标准**:
  - [ ] 能从数据库加载学生列表
  - [ ] 支持下拉框选择学生
  - [ ] 调用 PlanningService 生成路线
  - [ ] 显示分析进度
  - [ ] 完成后显示结果摘要

### FR3: 结果展示完善
- **描述**: 完整展示规划分析结果
- **验收标准**:
  - [ ] 显示路线匹配列表
  - [ ] 显示三档目标（冲刺/稳妥/保底）
  - [ ] 显示匹配理由
  - [ ] 显示行动建议

### FR4: 数据导出
- **描述**: 支持导出规划结果
- **验收标准**:
  - [ ] 支持 JSON 格式导出
  - [ ] 支持 Excel 格式导出
  - [ ] 文件内容完整正确

### FR5: 数据可视化
- **描述**: 使用图表展示分析结果
- **验收标准**:
  - [ ] 显示匹配度条形图
  - [ ] 显示分数对比图
  - [ ] 图表可交互

## 非功能需求

### NFR1: 性能
- Excel 导入支持最多 1000 行数据
- 规划分析响应时间 < 10 秒
- 页面加载时间 < 3 秒

### NFR2: 可用性
- 清晰的错误提示
- 进度可视化
- 操作反馈及时

### NFR3: 可靠性
- 导入失败时回滚事务
- 异常情况下不损坏数据
- 支持重复导入（去重）
