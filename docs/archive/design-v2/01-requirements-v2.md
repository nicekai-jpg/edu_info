# 01 - 需求分析（修订版 v2.0）

## 1. 核心定位转变

### 1.1 从"高考志愿填报工具"到"长期生涯规划系统"

| 维度 | 传统工具 | 本系统 |
|------|----------|--------|
| **时间跨度** | 高考前3个月 | 初中到大学（5-10年） |
| **核心目标** | 分数匹配院校 | 识别未来趋势+制定路径 |
| **用户群体** | 高三学生 | 初中生及家长 |
| **规划性质** | 被动响应 | 主动预判 |
| **输出形式** | 志愿表 | 三档目标+行动方案 |

### 1.2 提前规划的价值

```
初中阶段（7-9年级）：
├─ 发现兴趣特长
├─ 培养基础能力
├─ 了解多元升学途径
└─ 初步确定方向

高中阶段（10-12年级）：
├─ 深化学科优势
├─ 参与竞赛/科研
├─ 积累综合素质
└─ 精准定位目标

成果：
✓ 不是"考多少分报什么学校"
✓ 而是"想成为什么样的人，选择什么路径"
```

## 2. 用户画像（详细版）

### 2.1 主要用户：初中生家长

```yaml
基本信息:
  孩子年龄: 12-15岁（初一至初三）
  家长年龄: 35-45岁
  教育意识: 较强，愿意提前规划
  信息能力: 中等，需要工具辅助分析

核心诉求:
  1. 了解未来10年哪些行业/专业有前景
  2. 根据孩子特点选择合适的培养路径
  3. 提前准备多元升学途径（特长生、竞赛等）
  4. 制定可执行的长线规划
  5. 灵活调整，降低风险

痛点:
  - 不了解国家发展战略
  - 不知道有哪些升学途径
  - 不清楚各个专业的真实情况
  - 无法评估孩子的潜力
  - 担心选择错误耽误孩子

使用场景:
  - 初一时：确定大方向（科技/艺术/常规）
  - 初二时：选择特长培养（信息学/机器人/美术等）
  - 初三时：确定高中目标（重点/特色/国际）
  - 高中期间：持续跟踪调整
```

### 2.2 次要用户：初中学校教师/升学顾问

```yaml
角色: 初中班主任、升学指导老师、教育咨询顾问

使用场景:
  - 为学生提供规划建议
  - 批量分析学生情况
  - 生成规划报告给家长
  - 跟踪政策变化
```

## 4. 数据需求（扩展版）

### 4.1 数据类型清单

#### A. 政策类数据

```
国家级政策:
  - 国家中长期教育改革和发展规划纲要
  - 十四五/十五五教育规划
  - 新课程方案（2022年版）
  - 新高考改革政策
  - 强基计划政策
  - 保送生政策
  - 特长生政策

省级政策（重点：辽宁）:
  - 辽宁省高考改革方案
  - 辽宁省中考改革方案
  - 辽宁省综合素质评价方案
  - 辽宁省特长生招生政策
  - 各市中考政策（沈阳、大连等）

特殊类型招生:
  - 高校强基计划简章
  - 高校综合评价简章
  - 高校专项计划简章
  - 高水平艺术团/运动队简章
  - 各高校特长生招生简章
```

#### B. 数据类

```
全国高校在辽宁录取数据:
  - 39所985高校在辽宁录取分数
  - 116所211高校在辽宁录取分数
  - 147所双一流高校在辽宁录取分数
  - 其他重点高校在辽宁录取分数

分数线数据:
  - 2021-2025年各专业录取分数线

```

## 5. 非功能需求

### 5.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 单次分析时间 | < 5秒 | 生成完整规划方案 |
| 数据导入时间 | < 30秒 | 导入1000条分数线 |
| 启动时间 | < 3秒 | 程序启动 |
| 内存占用 | < 1GB | 运行时内存 |

### 5.2 可用性需求

```
离线可用:
  - 完全离线运行
  - 不需要网络连接
  - 数据本地存储

易用性:
  - 图形界面操作
  - 无需编程知识
  - 清晰的向导流程

可维护性:
  - 数据易于更新
  - 算法可调整参数
  - 模块化设计
```

### 5.3 安全需求

```
数据安全:
  - 学生信息本地存储
  - 不涉及网络传输
  - 数据备份机制

隐私保护:
  - 学生信息脱敏选项
  - 本地加密存储（可选）
```

## 7. 用户场景示例

### 场景1：初一学生初步规划

```
背景:
  学生：初一，成绩中上，对编程感兴趣
  家长：想了解科技特长生的路径

系统操作:
  1. 创建学生档案（初一、沈阳、成绩B+、兴趣：编程）
  2. 运行趋势分析 → 识别出"人工智能"为突破口
  3. 查看相关政策 → 发现"科技特长生"途径
  4. 匹配专业 → 计算机类、电子信息类
  5. 匹配高校 → 东北大学、大连理工（高/中/低三档）
  6. 查看可行性 → 需要提升数学+开始学习信息学竞赛
  7. 生成报告 → 初升高目标：省实验中学科技班

输出:
  - 三档目标高校
  - 科技特长生准备计划
  - 信息学竞赛学习路径
  - 初中三年行动清单
```

### 场景2：初三学生确定方向

```
背景:
  学生：初三，成绩优秀，已获得信息学省二奖项
  家长：想了解强基计划是否可行

系统操作:
  1. 更新学生档案（添加竞赛奖项）
  2. 分析强基计划政策
  3. 匹配强基计划高校和专业
  4. 评估录取概率
  5. 生成备选方案

输出:
  - 强基计划目标高校（冲刺）
  - 高考统招目标（保底）
  - 高中三年竞赛+学业规划
```
