## 1. 数据收集核心原则

```
✓ 高校范围：全国所有本科高校
```

## 4. 规划路线穷举体系

### 4.1 路线分类

| 路线类别 | 子路线数量 | 适用对象 | 准备周期 | 成本范围 |
|---------|-----------|---------|---------|---------|
| 普通高考 | 6条 | 所有学生 | 3年 | 1-10万 |
| 科技特长 | 10+条 | 有编程/科创兴趣 | 2-5年 | 3-15万 |
| 艺术特长 | 8条 | 有艺术天赋 | 5-10年 | 5-30万 |
| 体育特长 | 6条 | 有体育特长 | 5年+ | 3-15万 |
| 拔尖人才 | 10+条 | 学科能力突出 | 3-5年 | 2-15万 |
| 特殊类型 | 6条 | 特定需求 | 不定 | 5-50万 |
| **总计** | **50-100条** | | | |

### 4.2 路线匹配维度

```python
# 5维度匹配评分
weights = {
    'interest': 0.40,    # 兴趣匹配（最重要）
    'ability': 0.25,     # 能力匹配（学科+特长）
    'economic': 0.20,    # 经济匹配（路线成本vs家庭收入）
    'time': 0.10,        # 时间匹配（准备周期可行性）
    'region': 0.05       # 地域匹配（目标地区偏好）
}

def match_score(route, profile):
    """计算路线匹配度"""
    interest_score = calculate_interest_match(route, profile.interests)
    ability_score = calculate_ability_match(route, profile.scores, profile.talents)
    economic_score = calculate_economic_match(route.cost, profile.family_income)
    time_score = calculate_time_match(route.period, profile.grade)
    region_score = calculate_region_match(route.regions, profile.preferred_regions)
    
    return (
        interest_score * 0.40 +
        ability_score * 0.25 +
        economic_score * 0.20 +
        time_score * 0.10 +
        region_score * 0.05
    )
```
