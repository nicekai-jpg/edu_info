# 院校专业全量爬虫系统设计文档 (Design Spec)

**日期**: 2026-04-20
**状态**: 已批准
**目标**: 实现对 173 所重点高校（全国 985/211 及辽宁全量本科）专业数据的自动化、高稳固性抓取。

## 1. 业务目标
- **真实性**: 替换当前的模拟数据，获取教育部备案的真实专业列表。
- **完整性**: 抓取所有 173 所目标院校的完整专业库。
- **合规性**: 采用极致慢速与模拟真人行为，确保不被目标网站（阳光高考、辽宁招考办）封禁。

## 2. 技术架构
### 2.1 核心选型
- **引擎**: `Playwright` (浏览器模拟，最不易被封)。
- **插件**: `playwright-stealth` (抹除自动化特征)。
- **存储**: `DuckDB` (记录进度与最终结果)。
- **并发**: 顺序执行 + 分批重启 (Session Shuffling)。

### 2.2 抓取源策略
- **源 A: 阳光高考**: 抓取专业元数据（名称、学制、学位、门类）。
- **源 B: 辽宁招考办**: 抓取辽宁招生计划（专业代码、招生人数、学费）。

## 3. 稳固性与反爬加固设计 (Reinforced Design)
### 3.1 行为伪装
- **全路径模拟**: 从首页搜索院校名称，模拟真实的点击跳转，而非直接访问 URL。
- **非线性干扰**: 随机模拟鼠标滑动轨迹、页面滚动、无关空白处点击。
- **视口随机化**: 每次启动浏览器随机切换屏幕分辨率（如 1920x1080, 1440x900）。

### 3.2 礼貌抓取 (Politeness)
- **自适应延迟**: 翻页间隔随机 5-12 秒。
- **深度冷却**: 每爬取 3 所学校，强制停机休眠 5-10 分钟；每 10 所学校重启整个浏览器进程。
- **时间节律**: 模拟真人作息，非 24 小时高频运作。

### 3.3 零损耗恢复 (Zero-Loss Recovery)
- **进度表**: 建立 `scraping_checkpoints` 数据库表，记录每所学校的 `status` (pending, running, success, failed)。
- **断点续爬**: 启动时自动加载 pending 任务，确保不重复抓取。

## 4. 数据 Schema 扩展
### 4.1 专业表 (majors) 完善
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | INTEGER | 主键 |
| university_id | INTEGER | 关联院校 ID |
| name | VARCHAR | 专业名称 |
| category | VARCHAR | 学科门类 (如: 工学) |
| major_class | VARCHAR | 专业类 (如: 计算机类) |
| duration | INTEGER | 学制 (如: 4) |
| degree | VARCHAR | 授予学位 (如: 工学学士) |
| is_ln_recruiting | BOOLEAN | 是否在辽宁招生 |
| ln_code | VARCHAR | 辽宁招生专业代码 |

## 5. 实施路线图
1. **环境准备**: 安装 `playwright` 及 `stealth` 插件。
2. **原型开发**: 实现单所学校的搜索与解析逻辑。
3. **调度系统**: 开发断点续爬调度器。
4. **全量运行**: 按照 985 -> 211 -> 辽宁省属的顺序分批次抓取。
5. **数据对齐**: 合并双源数据并入库。
